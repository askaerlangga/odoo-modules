X2many Search Bar

Instantly search, filter and group records inside any one2many or many2many list — no page reload needed.

Everything you need, right inside the list

Client-side — zero server round-trips

Smart Search

Type a keyword and pick a field from the dropdown to scope the search. Matched records appear instantly as a facet chip in the search bar. Search across Char, Text, Many2one and more field types.

Group By

Click the caret button next to the search bar to open the Group By menu. Records are sorted, grouped and collapsed by default so you can focus on the groups that matter. Click any group header to expand or collapse it.

Ordered Facets

Active filter and group-by chips appear as Odoo-style facets in insertion order. Remove any facet by clicking × or by pressing Backspace on an empty input — the most recently added facet is removed first.

Odoo-style search panel

Familiar UX — no learning curve

The search bar looks and behaves exactly like the native Odoo search panel. Typing a query opens a Search by dropdown listing all searchable fields. Select a field (or press Enter) to commit the search as a coloured facet chip. Use ArrowDown / ArrowUp to navigate the suggestions, and Escape to close the dropdown.

Only records already loaded on the current page are filtered — making the experience instant and completely offline.

Client-side Group By

Collapse, expand and switch groups on the fly

Opening the Group By menu (▾ caret button) lists all configured fields. Selecting one immediately sorts and groups the visible records. Each group header shows the group label and a record count badge. Groups start collapsed so the view stays clean; click any header to expand or collapse it individually.

Combine search and group by simultaneously — the widget applies the filter first, then groups the matching results.

Section & Note support

Works on Sales Orders, Invoices and more

When the Accounting module is installed, the widget automatically inherits full Section & Note editing capabilities — no extra configuration or separate module required. Add a section, add a note, and fill in the description exactly as you would with the default widget.

The upgrade happens transparently via the fields registry at runtime, keeping ae_x2many_search free of any hard dependency on account.

Quick Setup

Add the widget to any one2many or many2many field

Add widget="x2many_search" to any one2many or many2many field in your view XML and list the fields to search/group via the search_fields option:

<field name="order_line" widget="x2many_search" options="{'search_fields': ['product_id', 'name', 'product_uom']}"/>
Options
  • search_fields (list, optional) — field names to search and group by. Matched case-insensitively as substrings. For many2one fields the display name is used. If omitted, all visible columns are used automatically.
Keyboard Shortcuts
  • Enter — commit search (auto-selects first field)
  • ↓ / ↑ — navigate field suggestions
  • Escape — close dropdown
  • Backspace — remove last facet (empty input)

Notes & Compatibility

  • All filtering and grouping is client-side only — only records loaded on the current page are affected (default page size: 80 records).
  • Supports both one2many and many2many field types.
  • All standard widget options (editable, create, delete, no_open, etc.) continue to work alongside the search options.
  • Section & Note editing requires the Description column to be visible (optional columns menu). This is standard Odoo behaviour.
  • Compatible with Odoo 17 Community and Enterprise.
